# Security Policy

## Supported Versions

Security fixes are applied to the latest code on `main` and the most recent
released tag.

| Version | Supported |
| --- | --- |
| Latest release | Yes |
| Older releases | No |

## Reporting a Vulnerability

Do not open public issues for suspected vulnerabilities.

Instead:

1. prepare a minimal reproduction and impact summary
2. describe affected versions and deployment assumptions
3. send the report privately to the project maintainers through the repository
   security contact or another agreed private channel

The project should acknowledge receipt quickly, investigate, and coordinate a
fix and disclosure timeline before public release.
